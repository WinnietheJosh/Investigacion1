¿Qué es? Importancia Principios

¿Qué es?

Se trata de la comodidad, utilidad y y facilidad que debe poseer una página web para los usuarios.

Una página web debe ser intuitiva, sencilla y agradable; de manera que cada usuario pueda acceder y usarla de forma certera sin necesidad de alguna guía.

Importancia

Las páginas web que a simple vuste, parecen complejas, con mal diseño, y con confuso funcionamiento, un usuario optaría por abandonar rápidamente esa página web.

Por lo que, procurar una página agradable, cómoda y de fácil uso, podría hacer al usuario realizar funciones de la web, como una compra, una suscripción, etc...

Principios

Con esta base de la Usabilidad Web, Jakob Nielsen otorgó una serie de principios para solventar las páginas web mal vistas, a continuación, un resumen de ellas:

1- Visibilidad del estado del sistema

Mostrar en todo momento lo que sucede, enseñarle al usuario que la acción se concretó, mantener informado al usuario.

Ejemplo: Usbir un archivo a Drive, se muestra un mensaje de lo que pasa en el momento exacto.

2- Relación entre el sistema y el mundo real

Mostrar señales y palabras cotidianas en caso de algún fallo que pueda deberse a un error que el usuario no necesariamente debe conocer

Ejemplo: Una pantalla de "Página no encontrada", el usuario no debe saber que significa el "Error 404".

3- Control y libertad del usuario

En caso de algún paso en falso del usuario, es recomendable darle alguna posibilidad de deshacer lo que hizo, remediar ese fallo.

Ejemplo: Borrar un mail, y mostrar la opción de deshacer esa acción.

4- Consistencia y estándares

Buscar la implementación de acciones lógicas, que el usuario pueda deducir que cosa es que, sin que tenga que buscar significados.

Ejemplo: Cambiar las típicas líneas horizontales de navegación, por cuatro cuadros.

5- Prevención de errores

Un error simple, que de igual manera se logre una solución a pesar del error que se pueda cometer, esto facilita y hace más cómodo toda la interfaz.

Ejemplo: En Google, a pesar de que falta una palabra, este lo deduce y muestra el resultado deseado.

6- Reconocimiento antes que recuerdo

Dejar todo al alcance del usuario, para que no tenga que recurrir al recuerdo, si no, a la visión.

Ejemplo: Historial de reporddución en Youtube.

7- Flxibilidad y eficiencia de uso

Atajos o diferentes aceleradores para mejorar la eficacia de la página y comodidad de usuarios antiguos y nuevos.

Ejemplo: Exchange ofrece una opción de configuración avanzada para este mismo principio.

8- Estética y diseño minimalista

Lo simple es mejor, solo mostrar información necesaria en cada una de las interfaces, menos es más.

Ejemplo: La página oficial de Apple, solo muestra lo necesarios, sus servicios, sus productos y nada más.

9- Ayudar a los usuarios a reconocer, diagnosticar y recuperarse de errores

Mostrar mensajes de error de manera clara, que el usuario entienda que debe hacer en caso de algún error en la página.

Ejemplo: Algún registro en una página, que determine que se debe repetir la contraseña, o que no coinciden; un mensaje claro para resolver dicho problema.

10- Ayuda y documentación

Dejando de lado que la página web debería de ser intuitiva y sin ninguna guía, sería útil alguna ayuda sobre algunos pasos, o de lo que sea necesario.

Ejemplo: Un inicio de sesión en cualquier página, la página puede dar alguna ayuda sobre en que se está fallando, en caso de fallar.
